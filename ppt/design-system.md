# Olares × Intel PPT 设计规范

> Slide 10–14 的视觉单一事实源。设计新页或修改旧页前先读；内容事实与来源仍由各页 Markdown / references 管理。

## 1. 画布与安全区

- 画布：`1280 × 720`，16:9。
- 页面背景：纸白 `#FAF9F6`。
- 横向安全边距：`44px`；顶部 `24px`；底部 `18px`。
- 深色侧栏可以延伸到右边缘，但正文仍从安全区开始。
- 禁止滚动、裁切和超出画布。

## 2. 字体与层级

字体栈：

```css
font-family: Inter, "Helvetica Neue", Arial, "PingFang SC", sans-serif;
```

- 主标题：`28px / 1.12 / 800`，优先 2–6 个词，只表达本页主题。
- 副标题：`14px / 1.4 / 400–700`，承载完整观点、结论或价值主张。
- 表头：`11.5px / 800`，大写，字距 `0.6px`。
- 表格正文：`12.5px / 1.36`。
- 方案名 / 行标题：`13.5px / 800`。
- 侧栏正文：`11.5px`；侧栏重点：`12.5px / 700–800`。
- 标签、chips、次级参数：`10.5px`。
- 对外可见文字原则上不得低于 `10.5px`；来源、测试细节不应靠缩小字号塞进页面。

## 3. 标题规则

- 主标题短，副标题长；不要把一句完整结论硬塞成主标题。
- 标题尽量单行；需要换行时按语义断行，禁止仅为填满空间换行。
- 关键词可以使用 Intel blue，其余保持墨黑。
- 不使用页码页签、文件名、章节编号来占据标题区。

参考：

- Slide 10 主标题：`Where users already pay for the cloud — and Olares wins locally.`
- Slide 12 主标题：`External GPU for AI NAS`
- Slide 12 副标题：`One Arc Pro B70 turns the NAS you already own into a Personal Agent AI NAS.`

## 4. 颜色

```css
--ink: #141516;
--muted: #6B7076;
--line: #E7E6E1;
--paper: #FAF9F6;
--accent: #0068b5;
--accent-deep: #0a4a80;
--accent-soft: #e6f0fb;
--warning: #B07D2B;
```

- 蓝色只用于品牌关键词、重点数字、主方案和结构线。
- 深蓝用于连续侧栏，不拆成多张小卡。
- 金色只用于风险、限制和需要改进，不用于普通强调。
- 一页最多一个主强调色和一个风险色。

## 5. 间距

只使用 `4 / 8 / 12 / 16 / 24 / 32 / 44px` 间距尺度。

- 标题与副标题：`6–8px`。
- 标题区与正文：`8–16px`。
- 主内容区列间距：`24px`。
- 表格单元格：纵向 `6–8px`，横向 `8–10px`。
- 连续信息组：`16–24px`。
- 不能用 Grid 的比例行高制造空白；内容组应使用 `auto` 高度和明确 gap。

## 6. 表格

- 表格必须在标题、表头或底部短 caption 中说明“比较什么”。
- 数字口径保持一致；不同引擎、量化或测试形态放在方案名下一行。
- 表头不做无意义换行；列宽不足时先删冗余说明，再调列宽。
- 单元格只放决策需要的信息。完整价格口径、来源 URL 和测试边界进入 notes / references。
- 数字范围统一使用 `~x–y`；价格统一使用 `$`，增量价格使用 `+$`。
- 主方案可用浅蓝整行强调；优化上限或非直接横比数据用浅灰弱化。

## 7. 两种标准布局

### 全宽密集表格

参考 [Slide 10](./10/cloud-proven-scenarios-slide.html)。

- 标题 + 副标题位于顶部。
- 主体为全宽表格。
- 适合场景地图、市场证据和多列信息。
- 正文字号以 `12.5px` 为基准，不通过缩小字体增加列数。

### 主表 + 深色侧栏

参考 [Slide 12](./12/external-egpu-slide.html)。

- 左侧为主证据表，右侧为 Advantages / What needs work。
- 右栏各组使用内容自适应高度，紧邻排列。
- 侧栏是一块连续色面，不拆成卡片矩阵。
- 表格底部允许一行短 caption；禁止放来源路径或长免责声明。

## 8. 内容取舍

PPT 保留：

- 一眼能读懂的结论。
- 对决策有影响的价格、性能和限制。
- 必须当场说明的测试口径。

移入 notes / references：

- 来源 URL。
- “host / PSU excluded”等完整价格边界。
- pp512 / pp2048 / ctx_pp 的详细差异。
- 引擎兼容性、PR 编号和内部技术判断。

## 9. 禁止项

- 无意义页签、章节编号和装饰性 badge。
- 多层边框、零碎卡片和大量圆角框。
- 用颜色框住普通副标题或日期。
- 表格上方重复标题，或底部重复来源。
- 为填满页面拉开信息组间距。
- 把逐字稿、技术备注或免责声明塞进主画面。

## 10. 输出检查

- HTML 与 PNG 都按 `1280 × 720` 检查。
- 主标题、副标题、表格和侧栏没有溢出。
- 100% 缩放下能读清正文。
- 同一组幻灯的标题基线、左右安全边距、蓝色和字号一致。
- 修改 PPT 文案后同步对应 Markdown；详细依据同步 references / script。
