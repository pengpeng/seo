## 项目理解（前置）

Radarr 是 *arr 生态中负责电影端的自动化管理器：用户只需加一部电影，Radarr 就会监控 RSS 索引器、识别符合质量要求的资源、推送给 qBittorrent/SABnzbd 等下载客户端，并在下载完成后自动重命名和整理文件目录，最后通知 Plex/Jellyfin 刷新媒体库。它是完整 *arr 媒体栈（Sonarr+Radarr+Prowlarr+Jellyfin/Plex）的核心组件，也是前辈 CouchPotato 的事实接班人。

| 项目 | 内容 |
|------|------|
| 一句话定位 | 开源自托管电影集合管理器，监控/下载/整理电影的全自动流水线 |
| 开源 / 许可证 | 是，GPL-3.0 |
| 主仓库 | [github.com/Radarr/Radarr](https://github.com/Radarr/Radarr)（★ ~13.9k） |
| 核心功能 | RSS 监控自动抓片、自定义质量档（Custom Formats）、失败自动回退、Plex/Jellyfin 通知、多平台 Docker/NAS 支持 |
| 商业模式 / 定价 | 完全免费，社区捐赠支持；无付费档 |
| 差异化 / 价值主张 | *arr 生态核心环节、Custom Formats 精细质量控制、活跃社区（400+ 贡献者）、CouchPotato 官方推荐替代 |
| 主要竞品（初判）| Sonarr（同族 TV 端）、CouchPotato（前辈/已停维护）、Lidarr（音乐端）、Bazarr（字幕端）；整体栈竞品：Plex Pass 内置管理、Jellyseerr 请求层 |
| Olares Market | ✅ 已上架（`directions/market/apps.md` 第 163 行） |

## 核心洞见

1. **品牌护城河**：radarr.video 品牌词（`radarr`，9,900/月）KD 65，品牌护城河牢固——正面做品牌词无意义。但 radarr.video 本身是极简下载站，没有博客/教程/比较文，**所有教程型、对比型、集成型内容空间都开放给第三方**（现在由 trash-guides.info、servarr.com 主导）。

2. **可复制的打法**：radarr.video 自己没有内容打法——它完全依赖社区文档（servarr wiki）。**内容机会在"生态整合层"**：`radarr vs sonarr`（KD 7）、`home media server setup`（KD 18）、`radarr docker compose`（KD 19）这类词，当前排在 radarr.video 主页（内容薄弱），实际应该被专项指南拿走。

3. **对 Olares 最关键的词**：
   - **`radarr vs sonarr` / `sonarr vs radarr`（各 260/月，KD 7/16）**：*arr 生态用户决策词，Olares Market 同时有两者，天然写一篇双栈部署对比文，落 Olares 价值主张。
   - **`home media server setup`（110/月，KD 18，CPC $0.39）**：有商业信号，是自托管媒体全栈的入口词，Olares = Radarr+Sonarr+Prowlarr+Jellyfin 全套一键安装。
   - **`how to access radarr remotely`（20/月，KD 0）**：零 KD 长尾，直命中 Olares 内置 HTTPS 反向代理解决方案，是 GEO/FAQ 必收词。

4. **最大攻击面**：Radarr 本身是免费开源的，无闭源/收费痛点。用户最大痛点在**部署复杂度**（Docker Compose 配置、路径权限、容器网络、Synology 包更新滞后）和**远程访问配置**（Nginx/Tailscale 手工设置）。Olares 的价值主张正是"免配置、开箱即远程访问"——这是攻击面所在。

5. **隐藏低 KD 金矿**：
   - `radarr api`（KD 4）、`radarr api key`（KD 8）：开发者/集成词，KD 极低，量小（70/110）但 Olares API 层叙事可扩展。
   - `couchpotato alternative`（KD 0）：老用户迁移词，量虽小（20）但零竞争；CouchPotato 早已停止维护，Radarr 是社区公认的继任者，可并入 radarr alternative 文一起收割。
   - `is there a program like sonarr or radarr for music`（260/月，KD 18）：**被严重低估的高量词**——月量 260 在 radarr 生态里排前列，KD 仅 18，答案是 Lidarr，Olares Market 亦有 Lidarr，一篇文可同时拿到三个应用的品类词流量。

6. **GEO 前瞻布局**：
   - `run arr stack on personal cloud`：当 AI Overview 被问"哪里可以一键装 Radarr+Sonarr+Jellyfin"时，Olares 的答案要预先出现在高质量内容里。
   - `radarr vs plex media server`：Plex 是付费竞品，"免费自托管 Radarr+Jellyfin vs Plex Pass"是自然对比，GEO 前哨。
   - `how to run radarr without docker`：Olares 从 App Store 安装，不需要手写 Compose，完全命中这个隐性搜索意图。

7. **与既有 `olares-500-keywords` 的关联**：500 词表中未收录 Radarr 相关词（Radarr 在 500 词分析中无匹配记录）；`home media server`（1,300/月）类词是补充方向。`radarr vs sonarr`、`home media server setup` 是新增高价值词，可进入 Olares 内容 backlog。
