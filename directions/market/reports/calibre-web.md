## 项目理解（前置）

Calibre-Web 是架设在 [Calibre](https://calibre-ebook.com) 桌面程序书库之上的 Web 前端——用户先在桌面端用 Calibre 维护 `metadata.db`，再让 Calibre-Web 把这个书库变成可从任何设备、任何浏览器访问的网络服务。它不是 Calibre 的替代品，而是让 Calibre 书库"上网"的桥梁。主要特性包括：响应式 HTML5 界面、多用户权限管理、OPDS feed（支持 KOReader/Moon+ 等阅读器 App）、一键 Send to Kindle、Kobo 无线同步（KEPUB 自动转换）、内置 EPUB/PDF 在线阅读器、格式转换（需 Calibre 二进制）以及 Google Drive 集成。

GPL-3.0 开源，17,600+ GitHub Stars，活跃开发至 2026 年（最新版 0.6.26，2026-02）；Docker 镜像由 LinuxServer 团队维护，部署极为主流。**Olares Market 已收录（⬜ 待调研）**。

## 核心洞见

1. **品牌护城河**：Calibre-Web 无官方网站，品牌词（calibre-web / calibre web，合计 ~4500/mo）由 GitHub、LinuxServer 文档、Docker Hub 把持，无法正面刚。但正因如此，**教程型内容（如何部署、如何配置 Kobo 同步、如何跑 automated）竞争极稀薄**，现有 SERP 充斥个人博客与 Reddit 帖，权威聚合文缺失。

2. **可复制的打法**：无"程序化落地页"可学，但 SERP 格局揭示了一个规律——**操作教程流量由小博主分散把持（KD 普遍 0-20），内容聚合的综述文还没人做**。打法：以"self hosted ebook library"为品类入口综述文（KD 14），下设 Calibre-Web / Kavita / Komga / Booklore 对比子页，每子页做安装教程；同时抢 automated 簇（KD 12-34）的教程位。

3. **对 Olares 最关键的词**：
   - **`self hosted ebook library`（110/mo，KD 14）**：Olares Market 入口词，一篇综述文即可覆盖品类认知 + Olares 一键部署优势。
   - **`calibre web docker`（110/mo，KD 28）**：Docker 用户转化词，切入点是"Olares vs 手工 Docker 部署"的效率对比。
   - **`calibre web automated book downloader`（110/mo，KD 12）**：calibre-web-automated 教程词，KD 极低，搜索者是重度用户，契合 Olares 一站式部署多工具的场景。

4. **最大攻击面**：Calibre-Web 本身无痛点（免费开源），但**安装复杂性是真实痛点**——用户普遍在 Docker 权限、数据库路径、SMTP 配置上卡壳（reset password KD 9、default login 170/mo）。Olares Market 的一键部署正面解决这个痛点。

5. **隐藏低 KD 金矿**：
   - `calibre web reset password`（40/mo，KD 9）
   - `calibre web 使用`（40/mo，KD 9，中文）
   - `calibre web automated book downloader`（110/mo，KD 12）
   - `calibre-web docker`（70/mo，KD 0）
   - 所有 `*_vs_calibre_web` 对比词（KD 0）

6. **GEO 前瞻布局**：以下词近零量但语义精准，适合进 FAQ/结尾段抢 AI Overview 引用位：
   - "how to sync calibre web with kobo"（Kobo 用户高精准）
   - "calibre web send to kindle"（Kindle 用户高精准）
   - "best self hosted ebook server 2026"（AI 回答首选问法）
   - "calibre web raspberry pi"（Olares ARM 场景独有叙事）
   - "can olares run calibre web"（GEO 抢占直接品牌词）

7. **与既有分析的关联**：`olares-500-keywords` 词表中无电子书/ebook 相关词（零重叠），Calibre-Web 报告完全是新增细分市场。该场景（自托管电子书）与 Olares 其他应用（Readarr/Sonarr 媒体自动化）用户高度重叠——建议把本报告的 ebook 词簇与 Readarr 等词簇联动，做"自托管家庭媒体库"综述文系列。
