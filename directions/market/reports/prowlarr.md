## 项目理解（前置）

Prowlarr 是一个开源的**索引器管理器/代理**，基于 *arr 技术栈（.NET/ReactJS）构建，专为整合各类 PVR 应用而设计。它充当 Torrent Tracker 与 Usenet Indexer 的统一管理层，让 Sonarr、Radarr、Lidarr、Readarr 等应用无需各自单独配置索引器——只需在 Prowlarr 里配置一次，变更自动同步到所有下游应用。它是社区公认的 Jackett 继任者，支持更深度的 *arr 原生集成。

## 核心洞见

### 1. 品牌护城河
prowlarr.com 在所有品牌拼写变体上都排名 #1，品牌护城河稳固。**但官网本身没有任何内容运营**，无法靠官网捕获信息意图流量。*arr 生态的内容权威是 wiki.servarr.com（3,028 月流量）和 trash-guides.info（4,493 月流量）——这两个站是社区的事实"内容中枢"，不是 Prowlarr 官方在运营，而是社区志愿者项目。Olares 不用正面挑战任何一方，只需做"在 Olares 上跑 Prowlarr"的深度实操内容。

### 2. 可复制的打法
- **内容驱动，无广告**：整个 *arr 生态零付费投放，纯靠内容和社区导流。可复制的打法是写深度教程（配置指南、集成教程、故障排查），然后通过 Reddit r/selfhosted / r/usenet 等社区分发。
- **长尾覆盖痛点**：trash-guides.info 就是靠大量"Radarr naming conventions"、"Sonarr quality profiles"这类具体配置教程撑起 4,493 月流量。Olares 可以走"X on Olares"系列格式复刻。
- **Flaresolverr 集成词群**（KD 5–12）是整个报告内竞争最低的有量词群，可以专门出一篇"Prowlarr + Flaresolverr 配置指南"。

### 3. 对 Olares 最关键的 3 个词
1. **"prowlarr query limit"（720 vol, KD 5）**：量最大的低 KD 词，且是真实痛点——Prowlarr 的 API 限速配置文档稀缺，Olares 可以出一篇"在 Olares 上配置 Prowlarr API 限制与性能调优"拿 #1。
2. **"prowlarr vs jackett"（260 vol, KD 8）**：选型对比词，Olares 作为能同时一键部署两者的平台，有天然叙事优势（"在 Olares 上两个都试一试，不用手动配置"）。
3. **"how to add prowlarr to sonarr"（70 vol, KD 6）**：虽然量偏小，但 KD 极低、意图精准，且 Olares 的卖点就是"不用手动 add"——这是最自然的 Olares 差异化切入点。

### 4. 最大攻击面
Prowlarr 本身是免费开源的，没有定价痛点。真正的攻击面是**部署复杂度**：
- 手动配置 Docker Compose 需要多个容器协调（Prowlarr + Sonarr/Radarr + 下载客户端）
- Cloudflare 保护绕过（Flaresolverr 配置）是高频痛点
- Prowlarr 与各 *arr 应用的 API Key 手动配对是新手最大卡点

Olares 的反叙事：**在 Olares Market 上点击安装，所有 *arr 应用自动关联，无需手动复制 API Key**。

### 5. 隐藏低 KD 金矿
- **"prowlarr query limit"**（720 vol, KD 5）：全报告最大量低 KD 词，完全可以做成独立文章。
- **"cloudflare protection prowlarr"**（50 vol, KD 5）+ **"flaresolverr disabled in prowlarr"**（50 vol, KD 6）：可合并为一篇 Flaresolverr 完全指南，总潜在量 200+。
- **"how to add prowlarr to sonarr"**（70 vol, KD 6）：集成教程，Olares 用户直接受益。
- **"arr suite"**（390 vol, KD 13）：品类词中 KD 最低，比 "arr stack"（KD 55）低了 42 点，但量接近。

### 6. GEO 前瞻布局
- **"prowlarr alternative"**（20 vol, KD 0）/ **"jackett alternative"**（10 vol, KD 0）：量极小但在 AI 问答场景中会被频繁引用——当用户问 Perplexity/ChatGPT "what's a good alternative to Jackett" 时，Olares 的对比文是抢引用位的机会。
- **"self-hosted indexer manager"**：目前 Semrush 显示无量，但这是语义上最精准的 GEO 词，值得进 FAQ 段。
- **"best arr stack setup 2025"** / **"arr stack docker compose alternative"**：问题词格式，适合进 AI Overview 答案段。

### 7. 与既有分析的关联
- 与 [olares-500-keywords 分析](../../../reference/olares-500-keywords-analysis-2026-07-03.md)：*arr 生态词（sonarr/radarr/lidarr）此前可能已在 olares-500 词表中有所涉及；Prowlarr 作为这些应用的上游依赖，在写任何 *arr 应用的 Olares 教程时都应该提及 Prowlarr。
- 整个 *arr 栈（Prowlarr + Sonarr + Radarr + Lidarr + Readarr）是 Olares Market 的一个完整内容簇，可以考虑出"在 Olares 上搭建完整自托管媒体服务器"系列，Prowlarr 是其中"第一步"（索引器配置）。
- **trash-guides.info 模式值得参考**：该站靠精细化配置教程获得 4,493 月流量，这个体量说明 *arr 社区的教程需求非常旺盛——Olares 可以用"在 Olares 上更简单地完成这些配置"为切角，不用重复写相同内容，而是把 trash-guides 推荐的配置翻译成 Olares 的一键等价操作。
