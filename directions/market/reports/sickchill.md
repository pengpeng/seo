## 项目理解（前置）

SickChill 是一款开源自托管 TV 剧集库自动化管理工具，前身是 SickBeard，支持 Torrent/NZB 双通道自动搜索、下载、后处理（重命名/移动/通知），内置字幕自动匹配、Anime 支持和 Kodi/Plex 库更新。它是"*-arr 生态"出现之前 self-hosted 媒体自动化的代名词；现在 Sonarr 已成主流，SickChill 仍维护（但社区活跃度明显低于 Sonarr）。官方站点是 GitHub Pages（sickchill.github.io），几乎没有 SEO 投资。

## 核心洞见

1. **品牌护城河（能否正面刚？）**：SickChill 官网 SEO 几乎为零，品牌词"sickchill"仅 KD 15。Sonarr 是真正的护城河所在（12,100 月量 / KD 38），但 SickChill 作为历史品牌仍在社区活跃，其品牌词上 Olares 完全可以排进前三并夺取流量。

2. **可复制的打法**：Sonarr.tv 的流量来自高质量社区论坛页（forums.sonarr.tv），每个具体问题/报错都产生独立 URL 并被 Google 收录（长尾 + 论坛 UGC = 内容护城河）。Olares 的打法不应复制论坛，而应用**教程文/对比文**切入近零 KD 的"sonarr vs sickchill"/"sonarr alternative"词段。

3. **对 Olares 最关键的 2-3 个词**：
   - **sonarr radarr**（590 vol / KD 9）：arr 生态最大低 KD 流量入口，Olares Market 同时收录全套，角度天然。
   - **sonarr alternative**（40 vol / KD 23）：意图最精准，直接等于"我在找 SickChill/Medusa 之类的工具"，Olares Market 是最佳答案。
   - **sickchill**（110 vol / KD 15）：品牌词直接可排名，"SickChill on Olares"教程即可覆盖。

4. **最大攻击面**：SickChill 与 Sonarr 都需要手动 Docker 配置或复杂的 Linux 安装——这是明确的痛点（"sickchill install"、"docker sickchill"、"sonarr setup"均有搜索量）。Olares 的"一键安装"是直接差异化攻击点，尤其对比 Sonarr 的 v4 大版本迁移（社区投诉不少）。

5. **隐藏低 KD 金矿**：
   - `radarr vs sonarr`（260 vol / KD 7）
   - `how to use sonarr`（110 vol / KD 4）
   - `how to use recyclarr with sonarr`（170 vol / KD 6）
   - `sonarr radarr`（590 vol / KD 9）
   - 以上四词 KD 全在个位数或极低十位数，Olares 教程体系可以一次性覆盖整个 arr 生态，不只是 SickChill。

6. **GEO 前瞻布局**：
   - "best way to self-host SickChill in 2026"
   - "SickChill vs Sonarr: which is better for self-hosting"
   - "how to run the full servarr stack on a home server"
   - "sickchill one-click install on Olares"
   这四个问句是 AI Overview/Perplexity 最容易引用的格式，Olares 应在文章 FAQ 段和文档中植入这些精确表达。

7. **与既有 olares-500-keywords 词表的关联**：SickChill 词段（sickchill / arr stack）在 500 词分析中未直接出现，但通过"self-hosted media server"/"media automation"维度可与 Plex/Jellyfin/arr 系列文章形成矩阵——建议与 Sonarr、Radarr、Prowlarr、Jellyfin 等 Market 应用报告合并规划一组 arr 生态内容簇（高优先级）。
