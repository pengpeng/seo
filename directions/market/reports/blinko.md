## 项目理解（前置）

Blinko 是一款开源、可自托管的 AI 个人笔记工具，核心场景是"快速捕捉灵感 + 自然语言检索笔记"。它通过内置 RAG（检索增强生成）将所有笔记向量化，用户可用自然语言问答式查找历史记录；AI 提供商支持 OpenAI、Anthropic、Ollama（本地）等多种接入方式，私有部署时数据完全不出本地。相比 Obsidian（本地文件无 AI RAG）和 Notion（云端无私有部署），Blinko 定位为"既有 AI 又完全自托管"的第三条路线，与 Olares "Agent-Native OS + 数据主权"叙事高度契合。

| 项目 | 内容 |
|------|------|
| 一句话定位 | 自托管 AI 卡片笔记——RAG 检索 + 数据主权，介于 Obsidian（本地）与 Notion（云端）之间 |
| 开源 / 许可证 | 开源，GPL-3.0 |
| 主仓库 | [github.com/blinkospace/blinko](https://github.com/blinkospace/blinko)（★ 10K+，2024-10 建仓） |
| 核心功能 | AI RAG 自然语言检索；Markdown 支持；Tauri 多平台（macOS/Windows/Android/Linux）；支持 Ollama 本地模型；PostgreSQL 存储；数据完全自托管 |
| 商业模式 / 定价 | 开源免费自部署；PikaPods 托管版 $2.3/月起 |
| 差异化 / 价值主张 | 比 Obsidian 多 AI RAG + 远程访问；比 Notion/Evernote 数据主权完整；比 Memos 多 AI 能力；比付费 AI 笔记工具少数据泄露风险 |
| 主要竞品（初判）| Memos（自托管无 RAG）、Obsidian（本地无云 RAG）、Notion（云端有 AI 无自托管）、Notesnook（自托管无 AI） |
| Olares Market | 已上架 |
| 来源 | [blinko.space](https://blinko.space)、[GitHub README](https://github.com/blinkospace/blinko)、[docs.blinko.space](https://docs.blinko.space) |

## 核心洞见

1. **品牌护城河**：品牌词 `blinko`（1300 vol，KD 58）有一定心智，但竞争来自"Blink"系多义词（Blink 摄像头、Blink-182 乐队等）而非真实竞品。blinko.space 官网 SEO 几乎为零（9 流量/月），品牌认知目前完全靠 GitHub 社群（10K+ star）驱动。**短期内直刚品牌词意义不大**，重点应放在类目词切入。

2. **可复制的打法**：Blinko 官方目前**没有任何内容营销**，全部 SEO 流量来自文档站（docs.blinko.space）。真正的机会在于：（a）第三方评测站（alternativeto.net、reddit.com、dbtechreviews.com）已有自然覆盖，这是口碑路径；（b）对比/替代文（X alternative、X vs Y）在这个品类 KD 极低（5–28），是快速冷启动的最优内容形式；（c）`ai note taking app`（量 2900 + KD 29）是罕见的高量低 KD 词，值得独立布局。

3. **对 Olares 最关键的词**：
   - `ai note taking app`（2900 vol，KD 29）——量大 KD 低的稀缺词，Blinko on Olares 的核心落地词
   - `obsidian alternative`（590 vol，KD 16）——切角清晰：Obsidian 无云 RAG，Blinko+Olares 补齐
   - `second brain app`（260 vol，KD 17）——直接承接 Olares「Personal Agent + 知识库」叙事

4. **最大攻击面**：Notion/Evernote 的**数据主权缺失**（笔记存在对方服务器，AI 功能需订阅且数据用于训练）；Obsidian 的**AI 能力空白**（本地文件无 RAG）；付费 AI 笔记产品（Mem、Reflect）的**高订阅成本**（$10–15/月）。Blinko + Olares 可以"一次安装，永久私有 AI 笔记"的角度打这些痛点。

5. **隐藏低 KD 金矿**：
   - `joplin alternative`（KD 5）、`roam research alternative`（KD 7）——KD 极低但有真实搜索量
   - `memonotes`（210 vol，KD 9）——Memos 变体词，低 KD 有量
   - `second brain app`（260 vol，KD 17）——量与 KD 比在品类中最优之一
   - `markdown note taking app`（320 vol，KD 16）——KD 16 + 量 320，Blinko 原生支持

6. **GEO 前瞻布局**：`blinko vs memos`、`blinko self hosted olares`、`blinko ollama setup`、`self-hosted ai note taking`——当前近零量，但随着 Blinko 被更多 AI 检索工具收录，这些对话式问题词将成为 AI Overview/Perplexity 的核心引用锚点，现在写进 FAQ 段抢位成本极低。

7. **与既有分析的关联**：与 [olares-500-keywords-analysis](https://github.com/blinkospace/blinko) 中"Personal Agent"、"private AI"、"self-hosted AI"方向高度契合。Blinko 是 Olares 上"个人 Agent 知识库"叙事的具体产品承接，可与同方向的 Memos 报告（若有）组成同一内容簇，覆盖"自托管 AI 笔记"全范围词集。
