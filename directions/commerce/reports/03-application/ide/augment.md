## 项目理解（前置）

Augment Code 由前 Google 工程师 Igor Ostrovsky 创立于 2022 年，2024 年 4 月出 stealth，以 $252M 融资、$977M 估值（接近独角兽）横空出世。核心差异化是 **Context Engine**：对 40 万文件 / 多仓库做实时语义索引与结构化检索，理解"哪个模块调用哪个模块、什么已废弃"，而不是简单的 keyword grep——声称在相同模型下比 Claude Code 节省 33% token 成本。2025 年推出 **Cosmos** 统一智能体平台，覆盖 SDLC 全生命周期（triage → implementation → code review → incident response）。是当前 AI 编码赛道中企业安全合规最完备的产品之一（ISO/IEC 42001 + SOC 2 Type II）。

## 核心洞见

1. **品牌护城河稳固，无法正面硬刚，但最大对比词 KD 只有 9。** `augment code`（KD53）、`augment`（KD57）是典型高 KD 品牌词，Olares 不必竞争。但 `augment code vs cursor`（1,000 月量，**KD9**，CPC $7.97）是整份报告的最大惊喜——竞争几乎为零的高商业意图词，Olares 完全可以用一篇"三方对比（Augment / Cursor / 自托管）"页面占领。

2. **Augment 的内容打法：`/tools/` 程序化清单页 + `everything-claude-code` 热点蹭流量。** `/tools/8-top-ai-coding-assistants-and-their-best-use-cases` 用一篇中立清单文同时排名 `ai coding assistant`（9,900）、`ai code assistant`（8,100）、`best ai for coding`（9,900）等品类大词；`everything-claude-code` 页面（KD28，6,600 月量）是对 GitHub 热榜 repo 做内容蹭流量的典型——Olares 完全可以复制此模式，针对 Tabby、Aider、Ollama 等工具做"N 个最佳自托管 AI 编码工具"清单页。
