## 项目理解（前置）

Appwrite 是一套开源 Backend-as-a-Service（BaaS）平台，2019 年创立，定位为 Firebase 的开源替代方案，核心口号是"Build faster and scale bigger"。它将 Auth、Databases（文档模型）、Storage、Functions（Serverless）、Messaging、Realtime 和 Web Hosting 统一打包，提供一个 API 终点覆盖完整后端需求。与 Supabase（PostgreSQL 关系型）不同，Appwrite 采用 NoSQL 文档数据库，更贴近 Firebase 的开发体验。截至 2026 年 7 月主仓库 56,497 GitHub Stars，累计融资 $37M，**已在 Olares Market 上架**（一键自托管，开源自部署形态）。近期新增 Presences API 与 MCP Server 集成，跟进 AI Agent 趋势。

| 商业模式 / 定价 | 云：Free $0（2 项目，1 周不活跃暂停）/ Pro $15/项目/月（独立资源）/ Enterprise 定制；自托管完全免费无功能限制 |

## 核心洞见

1. **品牌护城河（能否正面刚？）**
   Appwrite 品牌词 `appwrite` 月量 6,600，KD=59——有一定心智但远小于 Supabase（品牌词月量 90,500）。品牌词无需攻，Olares 角度应攻"安装/部署"功能词和"firebase alternative"品类词。直接打 Appwrite 品牌词没有意义；利用其"已在 Olares Market"的身份，做"一键部署 Appwrite"的内容才是正道。

2. **可复制的打法**
   - **追热词博客**：Appwrite 运营写了 `claude 5`、`vibe coding tools`、`free hosting platform` 等热门话题文章，蹭趋势词引流（claude 5 一篇贡献 35 UV/月）；Olares 可复制这一打法，以"在 Olares 上部署 Appwrite + AI Agent" 追 MCP/Agent 词
   - **社区 Q&A `/threads/`**：用户提问 URL 偶发捕获长尾技术词（`localhost:5173`、`referenceerror crypto` 等），累计有几十条小流量——Olares 社区/论坛内容可同理积累
   - **不做付费广告**：完全内容驱动，对 Olares 同样可行，无需付费投放

3. **对 Olares 最关键的词**
   - `firebase alternative`（170 vol，KD17，CPC $6.94）：量最大 + KD 最低的品类词，Olares + Appwrite 组合的核心主攻词
   - `self-hosted firebase alternative` / `open source firebase alternative`（均 KD=0，量 20）：零竞争，精准自托管意图，Olares Market 直接覆盖
   - `appwrite self-hosted` / `install appwrite`（均 KD=0）：直接指向 Olares Market 一键部署体验

4. **最大攻击面**
   - **Free 层门槛**：项目 1 周不活跃即暂停（Supabase 是 7 天），限 2 个项目——对开发者摩擦大；自托管 Appwrite 无此限制，Olares 可强调"永不暂停、资源自有"
   - **Pro 定价 $15/项目**：按项目计费，多项目场景成本叠加，Olares 自托管一次性硬件成本更经济
   - **NoSQL 局限**：Appwrite 是文档型 NoSQL，不支持原生 SQL；有复杂查询需求的用户会转向 Supabase——这是 Appwrite 固有局限，Olares 同时支持两者可作为双保险卖点

5. **隐藏低 KD 金矿**
   - `appwrite mcp server`（KD=4，70 vol）：MCP 词极早期，几乎无竞争，AI Agent 集成场景
