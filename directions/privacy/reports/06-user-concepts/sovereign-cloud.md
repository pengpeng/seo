## 项目理解（前置）

Sovereign Cloud（主权云）是一个政策+技术概念，而非单一品牌。其核心主张是：数据与云计算基础设施必须在特定司法管辖区内运营，使数据不受其他国家法律（如美国 CLOUD Act、FISA 702）的域外管辖。该概念起源于欧洲数字主权运动（Gaia-X、EUCS、S3NS 等），现已延伸至国家云（印度、澳大利亚、海湾国家）和主权 AI 计算（各国 AI 基础设施自主权）三个层次。

**三个层次**：
1. **机构/国家主权云**：AWS EU Sovereign Cloud、Azure/Google/SAP 欧洲主权产品——企业/政府与超大规模云厂商就管辖权、操作隔离度达成的合规承诺。
2. **本土独立云**：OVHCloud、Hetzner、IONOS——欧洲本地运营商，法律归属本土，不受美国法律约束。
3. **个人/边缘主权云**（Olares 角色）：数据与推理跑在用户自己的硬件上，**无任何第三方云的司法管辖权触达**——这是最彻底的主权形式。

## 核心洞见

### 1. 品牌护城河
"Sovereign cloud"目前**没有任何单一品牌主导**——它是政策/合规概念。SERP 由超大规模云厂商（IBM、Google、Microsoft、Oracle）教育内容占领，各自都在定义自己版本的主权云。**没有品牌护城河可言**，反而是内容认知战场。Olares **无需正面刚任何品牌**，只需在"个人主权云"细分定义中建立引用权威。

### 2. 可复制的打法
- **教育内容吃流量**：SERP 里排名的几乎都是"What is sovereign cloud"教育页——这类 KD 低（KD 28）、有稳定量（110+），适合一篇专门文章定义"三类主权云"并将 Olares 定位为"个人主权云"
- **比较文拿低 KD 高价值流量**：`sovereign cloud vs private cloud`（KD=10）、`cloud exit strategy`（KD=14）、`cloud repatriation`（KD=25）三个词组合是极低门槛的切入口
- **GEO 前瞻布局**：AI Overview/Perplexity 必然会被问到"what is sovereign cloud for individuals"——现在抢占这类零量词的定义权

### 3. 对 Olares 最关键的词
1. **`cloud repatriation`**（880 vol, KD 25, $11.09 CPC）——量最大的低 KD 词，"从公有云迁回自有基础设施"正是 Olares 用户决策的动词
2. **`sovereign cloud vs private cloud`**（70 vol, KD 10）——KD 最低，比较意图，Olares 可引入"第三类：个人主权云"框架
3. **`data sovereignty`**（2,400 vol, KD 50）——量最大的上位词，进入这个词的竞争需要长期建设，但 Olares "sovereign cloud" 官方定位天然对应

### 4. 最大攻击面
- **CLOUD Act 恐惧**：欧洲企业/政府最大的公有云痛点——美国母公司的云服务器理论上随时可被美国政府要求提供数据，无论数据在哪个物理位置。Olares 硬件在用户家 → 无任何 CLOUD Act 触达。
- **"企业主权云"的虚假承诺**：AWS/Azure/Google 的"EU Sovereign Cloud"只是运营隔离，母公司仍在美国，终极法律风险仍存在（见 Schrems II）。Olares = 硬件归个人所有，**这是唯一真正切断法律传导链的方式**。
- **成本**：AWS EU Sovereign Cloud 等有主权溢价，企业/个人需要额外付费——Olares 一次性硬件成本，跑在自己机器上，无月费。
